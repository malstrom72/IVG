# IVG Fiddle Zoom and Popup Implementation Plan

## Goals
- [ ] Deliver a user-controllable zoom system for the IVG Fiddle preview canvas that preserves raster fidelity while letting users inspect details at multiple magnifications without re-running compilation.
- [ ] Provide a background color popup that cycles through transparent, black, white, and the standard 16 web color keywords to simplify contrast and accessibility checks.

## Milestone 3 – Render Scaling Options and Pixel Fidelity
- [x] Expose a toolbar checkbox labeled "Vector rescale" (default off) that toggles between the current bitmap CSS scaling path and a new IVG vector-driven rescale workflow. The control should live next to the zoom dropdown, reuse the shared toolbar button styling, announce its state via `aria-pressed`, and serialize to `localStorage` under a new `STORAGE_KEYS` entry so `ZoomController.init` can hydrate it before binding events. (Inserted a toggle button adjacent to the zoom select, applied pressed-state styling, and persisted the preference via a new `ivgVectorScaling` storage key.)
- [x] Extend `ZoomController` (or a companion module) with a `setVectorScalingEnabled` flag and `isVectorScalingEnabled` accessor, making sure `applyZoom` consults the flag when deciding whether to mutate canvas transforms or trigger a reraster. Persist the preference on toggle and enqueue a debounced re-render so the canvas reflects the new mode immediately without requiring a manual zoom change. (Added vector-aware zoom math, `data-scaling-mode` image rendering hints, persistence hooks, and a requestAnimationFrame-backed rerender queue.)
- [x] When vector rescale is enabled, modify `runIVG` to rerun the IVG pipeline at the requested zoom factor by passing the clamped zoom multiplier into `rasterizeIVG` (e.g., `pixelRatio * zoomLevel`). Capture timing metrics in the trace log, guard against redundant renders while a compile is already in-flight, and note potential performance costs for large artboards directly in the plan comments for QA awareness. (Run pipeline with `pixelRatio * zoom`, log the active mode/scale, and gate concurrent renders with a queued rerun flag.)
- [x] Update the existing bitmap scaling branch to request nearest-neighbor interpolation by setting the canvas `style.imageRendering` to `pixelated` (plus `crisp-edges`/`-moz-crisp-edges` fallbacks) whenever vector rescale is disabled, and ensure any 2D context scaling uses integral scale factors where possible. Add a follow-up comment indicating which browsers ignore the hint so future contributors know the limitations. (Toggled a `data-scaling-mode="bitmap"` attribute that applies the fallback stack and annotated Safari’s lack of support in code comments.)
- [x] Extend manual QA notes to cover both scaling modes: verify the checkbox persists across reloads, confirm bitmap mode shows crisp pixels at 400%+, ensure vector mode maintains smooth curves without double-scaling artifacts, and recheck keyboard zoom shortcuts in each mode. (Documented QA expectations inline so reviewers cover persistence, 400% pixel checks, vector smoothing, and shortcut behavior for both paths.)
- [x] Run `timeout 600 ./build.sh` after implementing the scaling toggle to revalidate the build output. (Executed the full build/test matrix to confirm the vector toggle integrates cleanly.)
